# GJJ 开发工具文档

本目录存放 GJJ 项目中开发的通用工具和辅助功能的文档。

---

## 📚 工具列表

### 1. 依赖检查工具 (dependency_checker)

**文档**: [dependency_checker_usage.md](./dependency_checker_usage.md)  
**代码**: `nodes/common_utils/dependency_checker.py`

提供统一的依赖检查和错误提示机制，确保即使缺少依赖也不会导致整个节点包无法注册。

**功能**:
- `check_dependencies()` - 检查必需和可选依赖
- `create_fallback_node_class()` - 创建 fallback 节点类
- `safe_import_with_fallback()` - 安全导入模块并提供 fallback

**相关文档**: [dependency_fault_tolerance.md](./dependency_fault_tolerance.md)

---

### 2. 运行时依赖检查规范

**文档**: [runtime_dependency_check.md](./runtime_dependency_check.md)

**核心原则**: **在运行时检查依赖，而不是在模块导入时检查**。

**优势**:
- ✅ 节点可以正常注册，即使缺少依赖
- ✅ 用户在 ComfyUI 中能看到节点
- ✅ 执行时如果缺少依赖，会显示清晰的安装命令（含国内镜像）
- ✅ 彩色错误输出，更容易识别问题

**已应用节点**:
- Fish Audio S2 (`gjj_fish_audio_s2_loader.py`)
- CosyVoice3 (`gjj_cosyvoice3_runtime.py`)

## 🔧 使用场景

这些工具主要用于：

1. **节点开发**: 为新节点提供通用的辅助功能
2. **依赖管理**: 优雅处理缺失的 Python 依赖
3. **错误处理**: 提供友好的错误提示和降级方案
4. **代码复用**: 避免在不同节点中重复实现相同功能
