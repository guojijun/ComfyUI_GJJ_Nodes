# GJJ_PDFToImages

## 📋 概述

**功能**: 把 PDF 页面渲染为 IMAGE 批次，需要当前环境安装 PyMuPDF。

## 📁 文件映射

| 层级 | 文件 | 说明 |
|------|------|------|
| 🎨 前端 | `js/-` | 无前端文件 |
| 🔧 后端 | `nodes/gjj_pdf_tools.py` | `GJJ_PDFToImages` 后端执行逻辑 |

## 🔧 后端节点

### 基础信息

| 属性 | 值 |
|------|-----|
| **类名** | `GJJ_PDFToImages` |
| **CATEGORY** | `GJJ/PDF` |
| **FUNCTION** | `load` |

### 输入参数

| 参数名 | 类型 | 默认值 | 必填 | 说明 |
|--------|------|--------|------|------|
| `pdf_path` | `STRING` | `` | ✓ |  |
| `start_page` | `INT` | `1` | ✓ |  |
| `page_count` | `INT` | `1` | ✓ |  |
| `dpi` | `INT` | `144` | ✓ |  |

### 输出

| 输出名 | 类型 | 说明 |
|--------|------|------|
| 页面图片 | `IMAGE` | |
| 页数 | `INT` | |

## 🏗️ 数据流

```text
用户操作 → [前端扩展 UI] → ComfyUI 图引擎 → [后端节点执行] → 输出
         ↑ 参数设置/预览反馈
```
