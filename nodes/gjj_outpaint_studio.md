# GJJ · 🖼️ 多功能扩图工具

## 概述

GJJ 多功能扩图工具是一个集成了四种扩图工作流的单节点解决方案，支持 SD1.5、Flux1、Flux2 和 Qwen Image 等多种模型，提供一键扩图功能。

## 功能特点

- **自动模型匹配**：智能扫描模型目录，自动匹配可用模型
- **动态参数显示**：根据选择的扩图模式显示对应参数
- **批处理支持**：自动检测多图输入，批量处理
- **实时进度反馈**：执行过程中显示详细进度
- **一键下载**：模型缺失时提供下载链接和命令
- **内置预览**：生成结果即时预览，支持点击放大

## 支持的扩图模式

### 🎨 SD1.5 局部重绘

基于 Stable Diffusion 1.5 的局部重绘模型，适合简单扩图场景。

**模型要求：**
- `checkpoints`: 包含 `512-inpainting-ema` 的模型

**优缺点：**
| 优点 | 缺点 |
|------|------|
| 兼容性好 | 质量相对较低 |
| 速度快 | 细节保持一般 |
| 资源需求低 | |

**推荐参数：**
- 步数：25
- CFG：7.0
- 采样器：euler

---

### 🌀 Flux1 Fill 填充

基于 Flux1 填充模型，提供高质量扩图效果。

**模型要求：**
- `diffusion_models`: 包含 `flux1-fill-dev` 的模型
- `vae`: 包含 `ae` 的模型（如 `ae.safetensors`）
- `clip`: 包含 `t5xxl` 的模型
- `text_encoders`: 包含 `clip_l` 的模型

**优缺点：**
| 优点 | 缺点 |
|------|------|
| 生成质量高 | 资源需求高 |
| 细节丰富 | 速度较慢 |
| 边缘过渡自然 | |

**推荐参数：**
- 步数：20
- Guidance：3.5

---

### ✨ Flux2 Klein 增强

基于 Flux2 Klein 增强版，提供极高质量的扩图效果。

**模型要求：**
- `diffusion_models`: 包含 `flux-2-klein-9b` 的模型
- `vae`: 包含 `flux2-vae` 的模型
- `text_encoders`: 包含 `qwen_3_8b` 的模型

**优缺点：**
| 优点 | 缺点 |
|------|------|
| 质量极高 | 显存需求大 |
| 细节保留好 | 速度最慢 |
| 边缘平滑 | |

**推荐参数：**
- 步数：8
- Guidance：35

---

### 🌟 Qwen Image 智绘

基于通义千问图像编辑模型，支持智能扩图。

**模型要求：**
- `diffusion_models`: 包含 `qwen_image_edit` 的模型
- `vae`: 包含 `qwen_image_vae` 的模型
- `text_encoders`: 包含 `qwen_2.5_vl_7b` 的模型
- `controlnet`: 包含 `Qwen-Image-InstantX-ControlNet-Inpainting` 的模型
- `loras`: 包含 `Qwen-Image-Lightning` 的模型

**优缺点：**
| 优点 | 缺点 |
|------|------|
| 智能理解 | 配置复杂 |
| 语义准确 | 需要特定模型 |
| 支持多参考 | |

**推荐参数：**
- 步数：4
- Guidance：40

## 参数说明

### 🔧 扩图模式

选择扩图使用的工作流模式，共四种：
- `sd15_inpaint` - SD1.5 局部重绘
- `flux1_fill` - Flux1 Fill 填充
- `flux2_klein` - Flux2 Klein 增强
- `qwen_image` - Qwen Image 智绘

### ↔️ 扩图方向

选择扩图的方向范围：
- `all` - 四个方向同时扩展
- `left` - 仅向左扩展
- `right` - 仅向右扩展
- `top` - 仅向上扩展
- `bottom` - 仅向下扩展
- `left+right` - 左右扩展
- `top+bottom` - 上下扩展

### 📏 扩图像素

每个方向扩图的像素数，范围：8-512，步长：8

### 🎲 种子

随机种子，控制生成结果的随机性

### 👣 步数

采样步数，不同模式有不同的默认值

### ⚖️ CFG强度

分类器自由引导强度，控制提示词的影响程度

### 🌀 采样器

选择采样算法，如 euler、euler_ancestral、ddpm 等

### 📊 调度器

选择调度策略，如 normal、karras、exponential 等

### 🖼️ 输入图片

要扩图的原始图片，支持 `GJJ_BATCH_IMAGE` 和 `IMAGE` 两种类型，支持批量处理

## 使用步骤

1. **选择扩图模式**：根据需求选择 SD1.5、Flux1、Flux2 或 Qwen Image
2. **检查模型状态**：节点会自动检测所需模型是否齐全
3. **加载输入图片**：拖入或选择要扩图的图片，支持批量处理
4. **设置扩图参数**：选择扩图方向和像素数
5. **调整生成参数**：设置步数、CFG、采样器等
6. **执行扩图**：点击执行按钮开始扩图

## 故障排除

### 问题：模型列表为空

**解决方案：**
请访问 [模型下载地址](https://pan.quark.cn/s/6ec846f1f58d) 下载所需模型到对应目录。

### 问题：执行失败

**解决方案：**
检查控制台错误信息，确认所有模型都已正确安装。

### 问题：显存不足

**解决方案：**
尝试减小图像分辨率或使用较轻量的扩图模式（如 SD1.5）。

### 问题：扩图效果不佳

**解决方案：**
1. 尝试调整 CFG 强度和步数
2. 更换扩图模式
3. 调整扩图方向和像素数

## 模型匹配规则

1. **不分大小写**：`qwen`、`Qwen`、`QWEN` 等价
2. **不分子目录**：会搜索所有子目录
3. **最短优先**：文件名从短到长排序，优先匹配最短的
4. **包含子目录路径**：如果模型在子目录，列表会包括子目录路径

## 技术细节

### 输入输出类型

- **输入**：`GJJ_BATCH_IMAGE,IMAGE` - 兼容两种类型接入
- **输出**：`IMAGE` - 扩图后的图像批次

### 批处理模式

当检测到输入文件是多图时，自动走批处理模式，输出多张图片。

### 进度反馈

节点会实时发送进度信息到前端，包括：
- 模型加载状态
- 当前处理进度
- 执行耗时统计
