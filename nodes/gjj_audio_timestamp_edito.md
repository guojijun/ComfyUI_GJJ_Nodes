# GJJ · ✂️ 音频分段编辑器

## 功能简介

**GJJ Audio Segment Editor** 是一个功能强大的音频分段裁剪工具，支持节点内加载音频、自动生成分段、可视化编辑起止时间，并按时间段批量输出音频片段。

### 核心功能

- ✅ **节点内音频加载** - 内置音频文件选择器，无需外部节点
- ✅ **自动分段生成** - 根据音频时长自动创建等分时间段
- ✅ **可视化波形编辑** - Canvas高清渲染，拖拽调整起止时间
- ✅ **双标记系统** - 每个分段包含 start（开始）和 end（结束）时间
- ✅ **固定输出接口** - 最多支持99个音频片段输出
- ✅ **批量裁剪输出** - 一次性输出所有分段的音频片段
- ✅ **播放控制** - 内置播放器，实时预览调整效果
- ✅ **缩放视图** - 滚轮缩放波形显示（0.5x - 10x）

## 使用方法

### 1. 基本用法

#### 方式A：节点内加载音频
```
[选择音频文件] → [GJJ Audio Segment Editor] → [多个音频片段输出]
```

#### 方式B：外部连接音频
```
[Load Audio] → [GJJ Audio Segment Editor] → [多个音频片段输出]
```

### 2. 输入参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `外部音频` | AUDIO | 可选：外部连接的音频对象（优先级高于内部加载） |
| `音频文件` | COMBO | 节点内音频文件选择器（支持WAV/MP3/FLAC/OGG/M4A） |
| `分段数量` | INT | 自动分段数量（默认4，范围1-99） |
| `分段列表JSON` | STRING | 手动输入的时间段列表（可选，为空则自动生成） |

### 3. 分段列表格式

#### JSON 格式（推荐）
```json
[
  {"start": 0.0, "end": 2.5, "label": "片段 1"},
  {"start": 2.5, "end": 5.0, "label": "片段 2"},
  {"start": 5.0, "end": 9.12, "label": "片段 3"}
]
```

### 4. 交互操作

#### 鼠标操作
- **左键拖拽蓝色标记** - 调整开始时间
- **左键拖拽红色标记** - 调整结束时间
- **左键点击波形空白** - 播放/暂停音频
- **滚轮滚动** - 缩放波形视图
- **鼠标悬停标记** - 高亮显示当前分段

#### 右键菜单
- **🔄 重新生成分段** - 根据当前分段数量重新自动生成
- **➕ 添加新分段** - 在末尾添加新的时间段
- **🗑️ 删除选中分段** - 移除当前选中的时间段
- **📋 导出分段JSON** - 复制分段配置到剪贴板

### 5. 输出结果

| 输出 | 类型 | 说明 |
|------|------|------|
| `音频片段1...N` | AUDIO | 按时间段裁剪的音频片段（最多99个） |
| `分段列表` | STRING | 编辑后的时间段JSON配置 |

### 6. 输出接口说明

节点固定提供99个音频片段输出接口 + 1个分段列表输出：

- **实际使用**：根据分段数量，前N个输出接口会返回有效音频
- **未使用接口**：返回 None，不会连接到后续节点
- **动态感知**：前端会自动识别实际有效的输出数量

## 技术实现

### 后端 (Python)
- 文件: `nodes/gjj_audio_timestamp_editor.py`
- 依赖: `soundfile`, `numpy`, `torch`
- 功能: 音频加载、时间裁剪、批量输出

### 前端 (JavaScript)
- 文件: `js/GJJ_AudioTimestampEditor.js`
- 依赖: 无（使用浏览器原生 API）
- 功能: Canvas波形渲染、双标记交互、音频播放

1. **语音分段提取** - 将长语音自动分段，提取各个片段
2. **音乐片段裁剪** - 标记音乐的不同段落并分别输出

## 常见问题

### Q1: 音频不显示波形？
**A**: 
- 确认已选择音频文件或连接外部音频
- 检查浏览器控制台是否有错误
- 刷新页面（Ctrl+F5）清除缓存

### Q2: 输出接口数量不对？
**A**: 
- 检查分段列表JSON，确保包含有效的 start 和 end 字段
- 确认 start < end，且时间范围在音频时长内
- 实际有效的输出数量 = 分段列表的长度

### Q3: 音频片段时长不正确？
**A**: 
- 检查起止时间是否重叠
- 确认时间范围未超出音频总时长
- 查看控制台日志中的警告信息

### Q4: 如何实现真正的动态输出接口？
**A**: 
- ComfyUI 节点输出类型是静态定义的，无法运行时动态改变
- 本节点采用"固定最大数量 + None填充"策略
- 前端会根据实际分段数量自动识别有效输出
- 未连接的输出接口不会影响工作流执行
