# GJJ · ✂️ 可视化视频分段编辑器

## 概述

`GJJ_VideoSegmentEditor` 是一个强大的视频分段裁剪工具，支持在节点内加载视频、自动生成分段、通过Canvas可视化编辑起止时间，并按时间段批量裁剪输出多个视频片段。

## 核心功能

### 🎯 主要特性

- **节点内视频加载** - 内置视频文件选择器，支持 MP4、AVI、MOV、MKV、FLV、WMV、WebM 等格式
- **自动分段生成** - 根据视频时长自动创建等分的时间段（默认4段）
- **可视化编辑** - Canvas帧预览显示，拖拽调整起止时间标记
- **动态输出接口** - 根据分段数量自动扩展输出插槽（最多99个）
- **批量裁剪** - 一次性输出所有分段的视频片段

### 🎨 交互操作

- **左键拖拽蓝色/橙色标记** - 调整开始/结束时间边界
- **点击分段块** - 选中特定分段
- **滚轮缩放** - 查看时间轴细节
- **+ 添加按钮** - 在末尾添加新分段
- **均分按钮** - 将所有分段均匀分布到整个时长
- **删除按钮** - 删除当前选中的分段（至少保留1个）

## 使用方法

### 基本工作流

```
[Load Video] → [GJJ Video Segment Editor] → [Save Video / 其他处理节点]
```

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `video_file` | COMBO | 是 | 节点内视频文件选择器 |
| `video` | VIDEO | 否 | 外部连接的视频对象（优先级高于内部加载） |
| `segments_json` | STRING | 否 | 分段列表JSON，为空则自动生成 |
| `segment_count` | INT | 否 | 自动分段数量（默认4） |

### 输出

| 输出 | 类型 | 说明 |
|------|------|------|
| `分段列表` | STRING | 编辑后的时间段JSON配置 |
| `视频片段1...N` | VIDEO | 动态输出，数量等于分段数量 |

### 时间戳格式

```json
[
  {"start": 0.0, "end": 2.5, "label": "片段 1"},
  {"start": 2.5, "end": 5.0, "label": "片段 2"}
]
```

## 技术实现

### 后端处理

- **视频解码**: 使用 `imageio` + `pyav` 或 FFmpeg 命令行提取帧
- **视频裁剪**: 优先使用 FFmpeg 进行快速分割（保持原始编码），回退到帧数组裁剪
- **帧率检测**: 通过 ffprobe 获取准确帧率
- **VIDEO对象**: 创建标准 ComfyUI VIDEO 对象，兼容后续节点

### 前端渲染

- **Canvas绘制**: 高性能时间轴可视化
- **拖拽交互**: 精确的时间边界调整
- **颜色管理**: 自动分配不同颜色区分分段
- **动画效果**: 平滑的UI响应

## 应用场景

### 1. 视频章节分离
将长视频按章节分段，提取各个片段用于独立处理或发布。

### 2. 精彩片段剪辑
标记视频的精彩时刻并分别输出，快速制作集锦。

### 3. 教学视频分段
按知识点时间戳分离教学内容，便于模块化学习。

### 4. 播客/访谈剪辑
按话题或发言人分段，方便后期处理和字幕添加。

## 故障排除

### 问题1: 输出接口数量不对
**解决方案**: 检查分段列表JSON，确保包含有效的 `start` 和 `end` 字段。

### 问题2: 视频片段时长不正确
**解决方案**: 检查起止时间是否重叠或超出视频总时长。

### 问题3: FFmpeg未找到
**解决方案**: 
1. 确认已安装FFmpeg
2. 检查PATH环境变量是否正确配置
3. 在终端运行 `ffmpeg -version` 验证

### 问题4: 视频无法加载
**解决方案**:
1. 确认视频文件格式受支持
2. 检查文件路径是否正确
3. 尝试使用外部VIDEO输入而非文件加载

## 性能优化建议

1. **大视频文件**: 使用FFmpeg裁剪模式（有原始文件路径时自动启用），速度更快
2. **多分段处理**: 建议单次不超过20个分段，避免内存占用过高
3. **高分辨率视频**: 考虑先降分辨率再分段，最后重新合成
